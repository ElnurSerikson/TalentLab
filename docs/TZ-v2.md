## 0. Что меняется по сравнению с v1 (TL;DR)

v1 был «профориентация для школьников/студентов» = **одна аудитория × одна цель** (выбрать профессию).

v2 — это **self-profiler с автономными целевыми тестами на общей базе личности**:

1. **ICP расширяется на взрослых** (в поиске/пивоте и состоявшихся), при сохранении подростков/студентов.
2. **Архитектура: общая база личности + автономные глубокие тесты сверху** (pluggable-модули).
3. **Цель теста — это отдельный автономный тест, а не «линза» поверх одного отчёта.** Тесты рамочные (frame-of-reference): человек отвечает в контексте «найм» иначе, чем в контексте «строю своё», и это даёт более чистый результат.
4. **Прогрессивные промежуточные результаты** вместо похвалы: каждый этап заканчивается артефактом.
5. **Эпоха ИИ выходит на первый план** — и в вопросах, и в результатах.

Главный тезис: **сильные/слабые стороны — инвариант; арена успеха (учёба / работа / стартап / …) — переменная.** База меряет инвариант один раз; глубокие тесты меряют то, что зависит от рамки, и переиспользуют сигнал базы.

## 1. Аудитории (Ось А — «кто»)

Этап жизни захватывается в онбординг-анкете и **флейворит** все тесты (тон, примеры, применимость рекомендаций), но **не плодит отдельные тесты**.

| Сегмент | Кто | Что ищет | Как адаптируемся |
|---|---|---|---|
| `teen` | школьник 8–11 кл. | куда поступать, кто я | вузы/ЕНТ, дружелюбный тон |
| `student` | студент 1–3 курс | работа/специализация/стартап | стажировки, первый опыт |
| `adult_search` | взрослый в поиске/пивоте | сменить путь, найти своё | пути рескиллинга, time-to-transition |
| `adult_settled` | состоявшийся взрослый | подтверждение + рост | валидация + «пара полезных советов» |

**Принципы адаптации под взрослых:**
- Убрать школьные рамки из вопросов (нет «перед экзаменом», «ЕНТ») → использовать механизм контекстных вариантов формулировок (он уже есть в движке).
- Карточки профессий для взрослых: вместо «вузы + ЕНТ» → «путь перехода, спрос, время на рескиллинг».
- Для `adult_settled` — режим «валидация»: акцент на «ты на своём месте, вот 2 точки роста».
- Рассмотреть **экспресс-режим** для взрослых (меньше терпения к 40-минутному тесту) — флаг на будущее, в v2 не обязателен.

## 2. Информационная архитектура и флоу

```
Регистрация (fake)
   │
   ▼
Онбординг-анкета            ← кто ты + зачем пришёл (этап жизни, контекст, ожидания)
   │
   ▼
БАЗА ЛИЧНОСТИ (10–15 вопросов)   ← frame-independent
   │
   ▼
«Твой характер» (мини-результат)  ← короткий, цепляющий, создаёт разрывы любопытства
   │
   ▼
ВЫБОР ТЕСТА (карточки)            ← 3 карточки на старте, расширяемо
   │
   ├──▶ Тест «Работа»    → Результат «Работа»
   ├──▶ Тест «Учёба»     → Результат «Учёба»
   └──▶ Тест «Стартап»   → Результат «Стартап» (фаундер-роль и т.д.)
   │
   ▼
Личный кабинет           ← база + все пройденные тесты, можно вернуться/пройти ещё
```

**Ключевые принципы флоу:**
- Глубокий тест **переиспользует** сигнал базы → он короткий (≈15–25 рамочных вопросов), а не повторяет темперамент/ценности заново.
- Результаты тестов **не противоречат** друг другу (общая база — якорь).
- Можно пройти несколько тестов; кабинет показывает все результаты.

## 3. Движок: frame-independent vs frame-dependent

Это ядро дизайна — что мерить в базе (один раз), а что в каждом тесте (в рамке).

### База (frame-independent, меряем один раз)
- **Темперамент** (энергия/темп/экстраверсия).
- **Стиль мышления** (аналитик / креатор / практик / социальный).
- **Ядро ценностей** (что важно).
- **Цифровая / ИИ-готовность** (базовый срез — кросс-сквозная тема 2026+).
- 1–2 стабильные черты (добросовестность, открытость).

→ Выход: профиль-снимок «Твой характер» + лёгкий архетип + ИИ-готовность + тизеры к тестам.

### Глубокий тест (frame-dependent, меряем в рамке)
То, что **меняется** в зависимости от контекста:
- мотиваторы и драйверы **в этой рамке**;
- толерантность к риску/неопределённости **в этом контексте**;
- предпочитаемая роль/позиция;
- поведение в сценариях, специфичных для пути.

### Формула результата теста
```
РезультатТеста = f(сигнал_базы, ответы_глубокого_теста)
```
Базовый сигнал нормализуется и подаётся как входной вектор; глубокий тест добавляет рамочные измерения и считает свою специфическую схему результата.

## 4. Модуль «База личности»

- **10–15 вопросов**, форматы разнообразные (шкалы + ситуации + выбор), чтобы не было монотонности.
- Меряет 5 доменов TalentLab (укрупнённо) + ИИ-готовность.
- **Результат «Твой характер»:**
  - короткий архетип (lite) + 3 ярких черты;
  - метрика «Готовность к эпохе ИИ»;
  - **разрывы любопытства, ведущие к тестам**: «Хочешь узнать, какой из тебя фаундер? на какой работе расцветёшь? куда пойти учиться? → выбери тест».
- Принцип: **цеплять, а не насыщать.** Это и хук, и единственный артефакт для тех, кто не пошёл в глубокий тест.

---

## 5. Абстракция «Тест» (pluggable-модуль)

Каждый тест описывается единой структурой, чтобы добавление карточки №4 было **контентом, а не пересборкой движка**:

```ts
interface TestDefinition {
  id: string;                 // 'career' | 'study' | 'startup' | ...
  title: string;
  card: { subtitle: string; icon: string; color: string; audiences: Audience[] };
  frame: string;              // рамка для формулировок вопросов
  questions: Question[];      // рамочные глубокие вопросы
  score: (base: BaseSignal, answers: Answer[]) => TestResult;
  ResultView: React.FC<{ result: TestResult }>;
}
```

Регистр тестов → хаб выбора рендерит карточки по доступным определениям и аудитории пользователя.

### 6.1 Тест «Работа» (рамка: найм)
По сути — рефактор career-логики v1 в модуль.
- **Меряет:** рабочие мотиваторы, предпочитаемую среду, роль в команде, интересы (RIASEC в рабочей рамке), поведение в рабочих сценариях.

- **Результат:** направления обучения + как ты учишься лучше всего + (teen) вузы/ЕНТ, (adult) форматы доучивания.

## 7. Тест «Стартап» (детально)

### 7.1 Новые измерения (которых нет/мало в базе)
- толерантность к неопределённости и хаосу;
- устойчивость к провалу (failure resilience);
- драйв продавать и рекрутить;
- визионер vs исполнитель;
- находчивость / resourcefulness;
- founder-market сигналы (одержимость проблемой, скорость итераций).

### 7.2 Модель фаундер-ролей (подтверждена)
Маппинг сильных сторон → роль в основании:

| Роль | Архетип | Кого ищем как кофаундера |
|---|---|---|
| **CEO** | Визионер — задаёт направление, ведёт, фандрейзит | сильного исполнителя (COO/CTO) |
| **CPO** | Продуктолог — придумывает и формирует продукт | дистрибуцию (CMO) |
| **CTO** | Строитель — реализует технологически | продукт/продажи |
| **CMO** | Гроусер — продвигает, растит | продукт/технологию |
| **COO** | Операционист — наводит порядок, масштабирует | визионера (CEO) |

### 7.3 Структура результата
- **Фаундер-роль** (одна доминирующая + вторичная).
- **Подходящие типы стартапов** под роль + интересы: B2B SaaS, consumer, marketplace, content/creator, services, deep-tech, **AI-native**.
- **Твоя суперсила** и **твой фатальный пробел** (фаундеры умирают на пробеле — напр. крутой продукт, не умеет продавать).
- **Какого кофаундера искать** (комплемент к пробелу).
- **Максимально честный вердикт** (по добру от Elnur): соло-фаундер / нужен кофаундер / **ты сильнее как мощный #2** / ранний сотрудник / пока не фаундер-материал. Без лести — это строит доверие у взрослых.
- **ИИ-угол:** какие AI-native паттерны стартапа тебе заходят.

> Конкретные *идеи* стартапов в v2 **не обещаем** (детерминированный движок → будет банально). Идеи — это LLM-слой следующей фазы. v2 даёт роль + типы + команду + вердикт.

## 8. Прогрессивные промежуточные результаты (ключевой фидбэк v1)

**Принцип: каждый этап заканчивается артефактом, а не «молодец, дальше».**

- После базы → карточка «Твой характер».
- Внутри глубокого теста → 1–2 мини-инсайта с **конкретикой** (не «продолжай», а «вижу в тебе сильный драйв к…»).
- **Свободная форма должна влиять/отражаться.** В v1 текст игнорировался — в v2 минимум: отражаем назад («Похоже, тебя драйвит X, тревожит Y») и используем теги для подстановки рамочных формулировок. (Полный NLP — LLM-фаза.)
- Движок уже умеет считать по частичным ответам → под-результаты по этапам технически дёшевы.

- **Разнообразие форматов:** добавить ситуационные кейсы и выбор-из-картинок, чтобы убрать монотонность («вопросы показались одинаковыми»).

## 10. Что переиспользуем из v1 / что переделываем

**Оставляем как есть:** стек, дизайн-система, UI-компоненты, математика скоринга (взвешенная сумма + нормализация), RIASEC, данные талантов (после ребаланса), компоненты визуализации отчёта (радар, карточки, прогресс).

**Перерабатываем:**
- Флоу/роутинг (анкета → база → результат → хаб → тест → результат).
- Сторы: `userStore` (+ поля аудитории/контекста), новый `answersStore` (по тестам), новый `resultsStore` (результаты по `testId`), отдельно базовый результат.
- Движок: разделить на `base` (общая база) + per-test scoring-модули.
- Отчёты: вместо одного большого отчёта — мини-результат базы + per-test результаты.
- Рекомендации: aware к аудитории и цели.

**Новое:**
- Модуль «База личности» + результат «Твой характер».
- Абстракция `TestDefinition` + регистр тестов + хаб-карточки.
- Тест «Стартап» + фаундер-модель + каталог типов + честный вердикт.
- Прогрессивные стадийные результаты.

## 11. Технические заметки

- **Сторы (Zustand persist):**
  - `userStore`: + `audience`, `context/goal`, профиль.
  - `baseStore`: ответы базы + вычисленный `BaseSignal` + результат «Твой характер».
  - `testStore` (мульти): `answers[testId]`, `index[testId]`.
  - `resultsStore`: `results[testId]`.
- **Движок:** `domain/base/` (расчёт базы) + `domain/tests/<id>/score.ts` (per-test). Общая математика выносится в `domain/scoring/primitives`.
- **Ребаланс покрытия:** обеспечить ≥4 вопроса на каждый из 60 талантов (с учётом базы + всех тестов).
- **Типы:** `Audience`, `BaseSignal`, `TestDefinition`, `TestResult`, `FounderRole`.
