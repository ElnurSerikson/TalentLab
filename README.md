# TalentLab

AI-профайлер и карьерный навигатор для школьников (8–11 класс) и студентов (1–3 курс).
Помогает понять не «кем быть», а **какой ты есть**: сильные стороны, подходящие
профессии и план развития.

> Это фронтенд-демо. Все данные моковые, бэкенда и AI-API нет — логика скоринга
> детерминированная и считается на клиенте. Данные пользователя хранятся локально в
> `localStorage`.

## Стек

- **React 18 + TypeScript** (строгий режим)
- **Vite** — сборка и dev-сервер
- **TailwindCSS 3** — кастомная палитра в `tailwind.config.ts`
- **react-router-dom v6** — роутинг + защита роутов
- **Zustand** (`persist`) — глобальный стейт (профиль, ответы, отчёт, фидбек)
- **react-hook-form + zod** — формы и валидация
- **framer-motion** — анимации и переходы
- **Recharts** — радар-диаграмма RIASEC
- **lucide-react** — иконки
- **Vitest** — юнит-тесты алгоритмов скоринга

## Скрипты

```bash
npm install        # установка зависимостей
npm run dev        # дев-сервер (Vite)
npm run build      # типчек + продакшн-сборка
npm run preview    # предпросмотр собранной версии
npm run typecheck  # только проверка типов
npm test           # юнит-тесты (vitest)
```

## Архитектура

```
src/
├── app/        # роутинг, layout, guards
├── pages/      # страницы (landing, auth, onboarding, screening, scoring, report, dashboard, feedback)
├── components/ # ui/ (база), test/ (движок теста), report/ (визуализация), layout/
├── data/       # контент: 60 талантов, ~45 профессий, вопросы, теги, архетипы, рекомендации
├── domain/     # types.ts + scoring/ (детерминированные алгоритмы)
├── store/      # zustand-сторы (user, test, report, feedback)
├── hooks/      # useLocalStorage, useScrollTop
└── lib/        # cn(), analytics-заглушки
```

## Модель TalentLab-60

60 талантов в 5 доменах: **Мышление**, **Творчество**, **Люди**, **Действие** и
**Цифровая адаптивность** (последний — ключевой для рынка 2026+). Каждый вопрос несёт
веса по талантам и по 6 осям RIASEC; отчёт собирается в `domain/scoring/buildReport.ts`.

## Пользовательский путь

Лендинг → регистрация (фейк) → онбординг (4 шага) → скрининг (35 вопросов) →
свободная форма → основной блок (50 вопросов с адаптацией формулировок) →
экран обработки → отчёт (9 секций) → личный кабинет / фидбек.

Можно войти как демо-пользователь «Айгерим» одной кнопкой на странице входа.

## Деплой
